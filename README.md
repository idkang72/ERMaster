### ERMaster

takahiro40264님의 최신환경 대응 버전을 Fork한 것입니다.
Update Site를 이용하여 Eclipse 2025-12R 버전에 설치하려고 하였으나
org.eclipse.gef.common 의존성 문제와 erd(XML) 파일을 읽는 라이브러리의
사용 문제로 오류가 발생하여 이 부분을 해결해 보고자 합니다.

Eclipse 플러그인 개발 경험이 전혀 없는 건 물론, EMF, GEF 등등의 패키지에 대한
경험도 없기에 잘 될지 모르겠습니다.

----

원 개발자님이 보실지는 모르겠지만, 

정말 잘 사용하고 있습니다. 일본과 한국이 프로그램 개발시 영어권과는 다른 언어에 있어
비슷한 고민이 있어서 그런지  한국어를 사용하는 입장에서 필요한 부분이 잘 구현되어 있습니다.
작은 규모의 서비스 개발시에는 이만한 툴이 없습니다.

이런 툴을 만들어 주셔서 감사하게 생각하고 있습니다.

적어도 Eclipse 2025-12R 버전에 동작하도록 수정된다면 이 README 파일을 갱신하도록 하겠습니다.

여담이지만, 본격적으로 비싸지 않은 상용 버전을 만들 생각이 없으신지 궁굼하네요.

----

#### (2026.02.27) 일단 된다! 버전

꼼꼼하게 테스트한 버전은 아니라서 조심스럽지만, 필수적인 몇 가지만 되는 걸 확인한 후 Prerelase(tag-0.0.7.1)로
올려 놓았습니다.

> [!WARNING]
> 구버전에서 작성한 파일을 읽었을 때 FK 컬럼이 제대로 표시되지 않는 문제가 발견되었습니다.
>
> 기존 Relation을 삭제하고 다시 연결하면 되는데, FK가 PK로 참여하고 있는 경우,
> 다른 하위 Relation이 있는 테이블에게까지 영향을 줘서 편집시 주의가 필요합니다.
> 
> 이 버전의 플러그인에서 신규로 작성한 파일은 괜찮습니다.

- 일괄 등록 대화창이 닫힐 때 사용했던 Image 객체 dispose() 호출(이건 원래 주석 처리되었던 걸 풀었습니다)
- ERM 파일 열 때, 모델 파일 생성일시, 갱신일시 속성 값이 없을 때 발생하는 오류 수정
  * 이전 버전의 Fork 버전으로 작성했던 파일을 읽을 때 오류가 발생하여서, 속성 값이 없으면 빈값("")을 사용하도록 하였습니다.
- 뷰편집 대화창의 SQL 편집 TextArea에 Eclipse 설정의 기본 글꼴 적용
  * 여담이지만, 뷰를 추가해서 편집할 수 있다는 게 좋네요, 이전에 사용했던 Fork 버전은 뷰가 있으면
    저장시 오류가 발생해서, 테이블로 추가하고 이름만 뷰로하여 사용하였었네요.
